Provider-aware PTY runtime

One exec contract for AI CLIs.

ai-e drives Claude, Codex, Gemini, Grok, Copilot, Kiro, and Antigravity through a modular Rust wrapper with shared output rules.

Install from npm Read the README 
npm install -g @bitkyc08/ai-e
ai-e claude "summarize this repo"
ai-e codex --model gpt-5-mini \
  "audit this package"
ai-e grok --model auto \
  --output-format stream-json \
  "explain the provider boundary"

Provider adapters stay explicit.

Interactive CLIs

Codex, Gemini, Grok, Copilot, and Claude can run through PTY supervision with session-tail output.

Headless providers

Kiro and Antigravity use safer one-shot command paths when PTY behavior is not reliable.

Common result shape

Text, json, and stream-json modes give callers one parsing surface across provider-specific CLIs.

Built for orchestration boundaries.

ai-e keeps permission flags, binary resolution, transcript discovery, and resume IDs inside provider adapters while preserving a stable stdout contract for cli-jaw and shell callers.

cli-jaw
  -> ai-e claude ...
  -> ai-e codex ...
  -> ai-e grok ...
  -> ai-e kiro ...
  -> ai-e agy ...
  -> ai-e gemini ...
  -> ai-e copilot ...

Release checks are explicit.

  • Rust format and compile: npm run fmt:check and npm run check.
  • Contract tests: npm run test covers Rust tests and npm package contracts.
  • Publish surface: npm run publish:dry-run verifies package contents before release.
  • Remote Pages state: this page is a local Pages entrypoint until an authorized push deploys `/docs`.
