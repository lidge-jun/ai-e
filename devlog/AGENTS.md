# Devlog Planning Rules

This file governs the `devlog/` tree.

- New durable phase documents must use numbered prefixes such as `00_overview.md`, `01_phase1.md`, and `02_verification.md`.
- Do not add bare `PLAN.md`, `plan.md`, or `RCA.md` files.
- Keep active planning under `devlog/_plan/`.
- Move a plan folder to `_fin/` only when the whole planning unit is complete.
